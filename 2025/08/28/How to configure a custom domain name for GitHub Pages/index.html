    How to Configure A Custom Domain for GitHub Pages

    Posted on 2025-08-28 Edited on 2025-08-29 Views: Word count in article: 318 Reading time ≈ 1 mins.

    Add DNS Records

    Find the DNS page on the website where you purchased your domain name. Here, using Cloudflare as an example, add the following records:

    Type Name Content
    A @ 185.199.108.153
    A @ 185.199.109.153
    A @ 185.199.110.153
    A @ 185.199.111.153
    CNAME www xxx.github.io (Replace xxx with your own GitHub id)

    Add Your Custom domain on GitHub Pages

    Another way is to add a CNAME file in the root directory of your GitHub Pages and write your new domain in it, as shown below. This is the same as filling in the custom domain on GitHub Pages above.

    If you use hexo g->hexo d to automatically generate pages, you will need to set this domain on GitHub Pages every time you regenerate and deploy it. Even if you add a CNAME file in the root directory of your GitHub Pages, you will need to re-add the CNAME file every time you automatically update the website, because the original files will be cleared during the automatic update. The solution is to put the CNAME file in the source folder of your Hexo websiteÂ  so that it will be automatically configured every time you update the website.

    Modify the Hexo Config File

    You can replace the URL in the _config.yml of Hexo with your new domain. This step is optional, and you can still access your website through the new domain without doing so.
